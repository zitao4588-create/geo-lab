# AI 曝光体检测量有效性验证协议

- 日期窗口：2026-07-13；消费端 64 次尝试在同一日内完成。
- 研究问题：当前四 provider 的离线 API 采样，是否足以代表对应消费端默认体验。
- API：严格复用当前 `sampleAiProviders` 路径，不启用联网、工具或引用增强。
- 消费端：每条提示词使用独立新会话，不追问、不纠正；记录是否自动联网、搜索过程和引用。
- 实体：两个外部目标产品、一个正对照、一个经公开检索未发现同名实体的虚构负对照。
- 规模上限：4 个实体 × 4 条提示词 × 4 个 provider，最多 64 个成功 API 回答。
- 安全停止：任何 fallback、连续授权错误、异常收费迹象或成功答案达到 64 条时停止扩张。
- 原始证据：完整保存在 `raw/api/` 与 `raw/consumer/`，评分不能替代原始回答。
- 判定：严格使用用户给定的绿色、黄色、红色规则；5/8 仅复测冲突提示词一次。

## 实际执行结果

- API：64 个主样本，63 个成功、1 个超时、0 次 fallback；全部未联网。
- 消费端：64 次独立新会话，57 条可评分最终回答、7 条生成或页面异常；保存 64 份页面证据与 64 张截图。
- 消费端有效回答中 33/57 显示自动联网；未手动开启搜索、深度思考或其他工具。
- 发生异常时保留原始状态，不追问、不纠正、不伪造补答。

## 核心解释边界

1. API 不等于消费端搜索。
2. 消费端单次回答不等于稳定排名。
3. 消费端自动联网而 API 未联网，属于测量机制差异，是主要指标的一部分，不作为噪声忽略。
4. 离线 API 的 P4 用于发现链接编造及已有知识中的来源认知，不解释为实时来源检索能力。
5. 本轮不修改生产代码、生产配置、限流、运行数据或公众号内容。
