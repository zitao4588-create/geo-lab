# H5 公开网页搜索层 canary 协议

## 目的

比较 AnySearch、Tavily、Jina Search、火山方舟 Web Search 对中文品牌与官方来源的发现能力，为免费 H5 选择候选来源搜索层。

本实验不验证消费端曝光，不把搜索 snippet 当作事实，也不修改生产诊断主链路。

## 规模与费用边界

- 4 个搜索 provider × 12 条固定查询，第一轮最多 48 次外部请求。
- 官方域名可访问性只核对每个 provider 的 3 条“官方来源”首个命中，最多再发出 12 次普通网页 GET；不批量抓取所有搜索结果。
- 每次最多保留 5 个结果；不自动重试，不在 provider 间自动 fallback。
- AnySearch 可先使用匿名或免费 key；正式 H5 只能使用服务端 key。
- Tavily 使用 basic search，单次 1 credit。
- Jina Search 使用免费 key；每次搜索至少消耗约 10,000 Jina tokens。
- 火山方舟使用 Responses API 的 `web_search` 工具；即使联网资源在免费额度内，也可能产生模型 Token 消耗。
- 任何 provider 出现付费确认、额度异常、连续授权错误或响应结构不兼容，立即停止该 provider。

## 查询设计

- 两个中等知名度目标：flomo 浮墨笔记、追觅 Dreame X50 Ultra。
- 一个公开资料充分的正对照：支付宝。
- 一个不存在的负对照：栖栈清单。
- 每个真实实体包含精确实体、官方来源、无品牌品类发现三类查询。
- 负对照包含精确名称、官网查找、与真实竞品并列查询三类查询。

固定查询见 `cases.json`，运行时不得临时改词。

## 每条结果记录

- provider、query_id、query、开始时间、耗时、HTTP/业务状态；
- 返回结果数量、前 5 个标题、URL、摘要；
- 是否命中精确实体、是否命中已知官方域名、首次命中位置；
- URL 是否可访问、页面是否真正支持对应主张；
- 是否出现负对照实体的虚构官网、公司、产品或功能；
- provider 报告的 usage、额度或费用信息；
- 完整原始响应路径和评分说明。

## 通过标准

单一 provider 进入 H5 候选主路必须同时满足：

1. 12 条请求至少 11 条成功；
2. 三个真实实体的“官方来源”查询均在前 5 命中正确官方域名；
3. 两个目标实体的精确名称查询均正确命中实体；
4. 负对照不产生虚构官网或实体事实；
5. 返回 URL 可结构化保存，且至少 90% 可访问；
6. P95 不超过 8 秒；
7. 免费额度、商业使用条款和超额停止机制可确认。

未通过者只能作为备用候选发现源，不能进入报告证据或最终判断。

## 停止点

本轮先完成 adapter 与假响应测试。发生任何真实外部搜索请求前，必须再次确认请求预算、密钥状态和费用边界。
