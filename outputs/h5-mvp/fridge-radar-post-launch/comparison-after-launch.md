## 3. 相比旧参考案例的取长补短

旧参考案例的优点仍然成立：模块完整、用户画像更细、行动建议像顾问交付。但旧案例的问题也更明显：多平台结果里包含虚拟/模拟口径，部分预测分数和提升幅度缺少真实采样支撑。

本版 H5 的改进点：

1. 真实采样边界更清楚：只把 DeepSeek 真实返回写成采样证据，其他平台明确 unavailable。
2. 公开基建不再靠主观判断：直接抓取 8 个线上 URL，记录 HTTP 状态、命中事实和缺失事实。
3. 报告可交付性更强：同一报告能导出 Markdown、HTML、PDF 和 evidence package。
4. 竞品模块更可复核：每个竞品都能追溯到具体问题和答案摘录。
5. 文案从“快速初筛”转为最终 GEO 分析成果，不再展示双分数。

仍然不如旧案例的地方：

1. 多平台真实采样还未接通，只完成 adapter 占位和证据边界。
2. 用户画像仍是规则生成，缺少真实用户搜索词、微信搜一搜词和站点访问数据。
3. 旧案例里的“外部内容分发/引用矩阵”还没有产品化成可检查任务清单。

## 4. 下一步优化方向

1. 接入第二个真实采样平台。  
   优先选择可以稳定 API 化、可保存原始回答、不会违反平台条款的平台。接入前不要把平台结果写成事实。

2. 为冰箱小雷达建立索引吸收观测。  
   每周固定检查 `site:fridge.playgamelab.cn`、品牌词搜索、llms.txt 可读性、sitemap 收录状态和 H5 复测结果。

3. 把站点内容补成可引用问答库。  
   在 FAQ 和功能页补“适合谁/不适合谁/为什么不用 AI 菜谱/为什么不默认上传图片/和鲜钟记、冰箱清单区别”等直接问答。

4. 增加外部引用证据。  
   用公众号、知乎、CSDN、GitHub README 或产品说明页互链到品牌站，但不要编造媒体背书、客户案例或排名数据。

5. 报告导出继续产品化。  
   下一版可以把 HTML/PDF 导出按钮直接放进 H5 结果页，并给 evidence package 增加一键下载入口和文件清单说明。

6. 把上线前后复测做成固定仪表。  
   当前已得到 61 -> 68 的第一组 before/after；后续每次内容改动都用同一 20 条 prompt 复测，观察提及率、风险词和竞品压制是否真的变化。
