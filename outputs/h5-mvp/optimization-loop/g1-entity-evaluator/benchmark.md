# G1 实体层级与判定 Benchmark

日期：2026-07-12

固定输入：

- `apps/ai-exposure-check-h5/test/fixtures/entity-evaluator-benchmark.json`
- `apps/ai-exposure-check-h5/test/fixtures/entity-hierarchy-ground-truth.json`

人工层级标注覆盖品牌、营销名、产品/服务、官方主型号、城市和门店六层。无法从输入可靠证明的层级明确标记为 `null` 或“未提供具体门店”，不由规则猜测。

| ID | 场景 | 关键断言 | 结果 |
| --- | --- | --- | --- |
| A01 | 完整商品名与官方主型号 | 完整名、营销简称、ES-LM55 均可正确绑定 | 通过 |
| A02 | 本地服务合理简称 | “海底捞”可作为完整业务名的实体别名；城市仍单独判断 | 通过 |
| A03 | 无品牌问题主动推荐简称 | 合理简称可计入自然推荐，通用品类词不可以 | 通过 |
| A04 | 只有通用品类词 | 不绑定实体，不计自然推荐 | 通过 |
| A05 | 错误主型号夹带正确相关型号 | 仍判错误认知，相关型号不能掩盖主型号错误 | 通过 |
| A06 | 正确主型号同时提及相关产品 | 主型号正确，可保留相关产品说明 | 通过 |

通用型号断言还覆盖 `ES-LM55`、`S9000`、`X-PRO3`，证明解析不依赖松下品牌前缀。

## Before / After

- Before：只有完整业务名才算品牌提及；“海底捞”无法代表“海底捞西安门店服务”。
- After：从业务名、城市和通用后缀派生安全别名；仍拒绝“火锅餐饮”“门店服务”等通用词。
- Before：回答同时出现 ES-LV9C 和 ES-LM55 时，只要包含一个官方型号就可能绕过冲突。
- After：优先解析“官方主型号/对应型号”声明；主型号错误仍判 misrecognized，相关产品不再掩盖。
