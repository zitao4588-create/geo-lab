# 报告可信度合同 v0.4

日期：2026-07-12

## 目标

报告结论强度必须与证据强度一致。系统不能再把表单完整度、品牌字符串复述、正确实体认知、无品牌词自然推荐和公开页面基建混成一个看似精确的结论。

## 输入预检

完整采样前先评估：

- `ready`：资料足以进入完整采样。
- `needs_confirmation`：需要补充或确认实体、行业、目标用户或官方入口。
- `insufficient_evidence`：证据不足，不调用模型、不消耗免费名额、不生成总分。

预检只使用用户输入和确定性规则，不调用模型。外部搜索结果只能作为 candidate；没有可靠 provider 时要求用户粘贴并确认官方入口，禁止模拟来源。

## 业务类型

- `physical_product`：型号、规格、适用场景、安全、售后、耗材、正规渠道。
- `software_or_miniprogram`：功能、用户场景、数据/隐私边界、访问入口。
- `local_service`：服务范围、城市、资质、预约、价格口径、售后。
- `generic_or_unknown`：必须先确认类型，不能强行套模板。

## 核心指标

- 字符串出现率：答案是否出现品牌字符串，只描述文本现象。
- 正确实体识别率：带品牌问题中，与已审计实体事实一致的回答比例；没有 ground truth 时显示 N/A。
- 无品牌词自然推荐率：问题未包含品牌名且模型主动推荐、同时实体识别有据的比例。
- 错误认知率：带品牌问题中被识别成错误型号、硬件/软件错误类型或其他实体的比例。
- 模型一致度：同一核心问题上占比最高的识别状态。
- 公开页面审计：只使用真实 PageAudit 结果；没有 URL 时显示“未检测”，不使用保底分。

## 评分

- `可信 AI 可见度 = 65% × 正确实体识别 + 35% × 无品牌词自然推荐`。
- 缺少已通过审计的公开来源时，保留内部兼容分值，但用户报告暂停展示总分。
- 旧版 `mentionRate` 继续保留用于历史兼容，但用户界面明确改名为“字符串出现率”。

## 报告第一屏

必须先显示：

1. 已确认事实。
2. 仍待核验内容。
3. 报告置信度。
4. 模型冲突。
5. 下一步三项行动。

Markdown、HTML、H5 和 evidence package 使用相同的 `stages.credibility` 数据源。

## 证据边界

- API 样本不是消费端搜索排名。
- 候选来源不是已验证事实。
- 不确定、猜测、非官方或无法确认的回答不计入正确识别。
- 问题中已有品牌名时，答案复述品牌不计入自然推荐。
- 不承诺排名、流量或推荐提升。
